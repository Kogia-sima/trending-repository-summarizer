# コード品質の自動改善

自動的にコードを修正してコード品質を保証します。

このコマンドは、以下の順序で品質チェックと修正を実行します：

1. **コードフォーマット** (`uv run ruff format`) - Ruffによる自動フォーマット
2. **リントチェック** (`uv run ruff check --fix`) - Ruffによるリントチェックと自動修正
3. **型チェック** (`uv run pyright`) - pyrightによる厳格な型チェック
4. **テスト実行** (`uv run pytest`) - 全テストの実行

## 実行手順

1. まず `uv run ruff format` コマンドでコードベース全体をフォーマット
2. エラーが発生した場合、自動修正可能なものは修正、手動修正が必要なものは詳細を報告
3. `uv run ruff check --fix` コマンドでリントチェックを実行
4. エラーが発生した場合、自動修正可能なものは修正、手動修正が必要なものは詳細を報告
5. `uv run pyright` コマンドで型チェックを実行
6. エラーが発生した場合、自動修正可能なものは修正、手動修正が必要なものは詳細を報告
7. `uv run pytest` コマンドでテストを実行
8. エラーが発生した場合、自動修正可能なものは修正、手動修正が必要なものは詳細を報告
9. 修正した点をまとめ、ユーザに報告

## 重要な注意事項

- **CLAUDE.mdの規約に従う**: 型ヒント、エラーメッセージ、コーディングスタイルはCLAUDE.mdの規約に準拠
- **既存の動作を保持**: テストが失敗した場合、まず実装が仕様通りかを確認し、必要に応じてテストも修正
- **段階的な修正**: 一度にすべてを修正せず、エラーの種類ごとに段階的に対処

## エラー対処の具体例

### 型エラーの例

```python
# エラー: Incompatible return value type (got "None", expected "str")
def get_name() -> str:
    return None  # 修正前

# 修正後
def get_name() -> str | None:
    return None
```

### リントエラーの例

```python
# エラー: F401 'os' imported but unused
import os  # 修正: 未使用のインポートを削除

# エラー: E501 line too long (120 > 100 characters)
# 修正: 長い行を適切に改行
```

このコマンドを実行することで、プロジェクトのコード品質を一貫して高いレベルに保つことができます。
