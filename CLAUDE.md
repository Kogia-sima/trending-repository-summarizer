## フォルダ構成

```
<project root directory>/
├── src/trending_repository_summarizer  # ソースコード
├── test/                               # テストコード
│   ├── unit/                           # ユニットテスト
│   ├── integration/                    # 統合テスト
│   └── testdata/                       # テストデータ
├── scripts/                            # ビルドやデプロイのスクリプト
└── README.md                           # プロジェクトドキュメント
```

## ルール

1.  パッケージ管理

    - **uvのみ**を使用し、pipは**絶対に使用しないでください**
    - 禁止事項: `uv pip install`、`@latest`構文

2.  コード品質
    - すべての関数の引数と返り値に型ヒントが**必須**です。
    - 公開関数にはdocstringが**必須**です。
    - インデントは必ずスペース4つを使用してください。タブ文字は使用しないでください。
    - コードの各ロジックに対して、日本語のコメントを追加してください。
    - 関数は単一の役割に集中し、小さく保つ必要があります。
    - 既存コードのパターンに忠実に従ってください。
    - docstringにはGoogleスタイルを使用してください。
    - 行の長さ: 最大100文字
    - 命名規則はPEP8に従ってください。
